# Madras Miles — Kiro CLI Context

## What is this?
Madras Miles is a walkathon challenge platform with Strava integration.
- Frontend: React/Vite/Tailwind at `/Users/epxxvis/projects/walky/client`
- Backend: Express/Prisma at `/Users/epxxvis/projects/walky/server`

## Deployments
- Frontend → Firebase Hosting: `https://madras-walkathon.web.app`
- Backend → Render (Docker, auto-deploys from GitHub `main` branch): `https://madras-miles-api.onrender.com`
- GitHub: `https://github.com/Vishwa310/madras-miles.git`

## Deploy Commands

### Deploy Everything
```bash
cd /Users/epxxvis/projects/walky
cd client && npx tsc --noEmit && cd ..
git add -A && git commit -m "MESSAGE" && git push origin main
cd client && npm run build && firebase deploy --only hosting
```

### Deploy Frontend Only
```bash
cd /Users/epxxvis/projects/walky/client
npm run build && firebase deploy --only hosting
```

### Deploy Backend Only (push triggers Render auto-deploy)
```bash
cd /Users/epxxvis/projects/walky
git add server/ && git commit -m "MESSAGE" && git push origin main
```

### Verify
```bash
curl https://madras-miles-api.onrender.com/api/health
curl -s -o /dev/null -w "%{http_code}" https://madras-walkathon.web.app
```

## Key Files
- `client/.env.production` — API URL, Strava keys (baked into build)
- `client/firebase.json` — Firebase config (public dir = dist)
- `server/Dockerfile` — Render builds from this
- `server/prisma/schema.prisma` — DB schema (auto-pushed on deploy)
